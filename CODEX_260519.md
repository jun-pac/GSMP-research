So yes: the original Elliptic transaction graph does not connect nodes with different timestamps. The timestamps are more like temporal batches/classes of transactions, while edges are within the same timestep in this version of the graph.

For SMP to matter on Elliptic, we would need to add a different graph construction step, for example cross-time similarity edges or temporal kNN edges. But that would no longer be “just applying SMP to the original Elliptic transaction graph”; it would be a new graph design choice.
