Implemented SMP for Elliptic Bitcoin as graph preprocessing only in [elliptic_bitcoin_smp.py](/home/jooyoungpak/workspace/GSMP-research/elliptic_bitcoin_smp.py).

What it does:
- Loads standard Elliptic CSVs: `elliptic_txs_features.csv`, `elliptic_txs_classes.csv`, `elliptic_txs_edgelist.csv`
- Computes the SMP condition per directed edge `u -> v`
- Doubles SMP “single” neighbors by duplicating those edges in `edge_index`
- Leaves the GCN operation unchanged: `GCNConv(x, edge_index)` only
- Adds temporal train/val/test masks using default Elliptic-style timesteps
- Includes a baseline GCN training/eval loop

Run vanilla GCN:

```bash
python3 elliptic_bitcoin_smp.py --root /path/to/elliptic
```

Run SMP-GCN:

```bash
python3 elliptic_bitcoin_smp.py --root /path/to/elliptic --use-smp
```

**Size Estimates**
Default model: 2-layer GCN, `165 -> 128 -> 2`

Parameter estimate:
- Layer 1: `165 * 128 + 128 = 21,248`
- Layer 2: `128 * 2 + 2 = 258`
- Total: `21,506` trainable parameters
- Weight memory: about `86 KB` in fp32
- Training memory for params + grads + Adam states: about `350 KB`

Standard Elliptic data estimate:
- Nodes: `203,769`
- Original directed edges: `234,355`
- Node features: `203,769 * 165 * 4 bytes ≈ 128 MiB`
- Directed `edge_index`: about `3.6 MiB`
- Undirected `edge_index`: about `7.2 MiB`
- Undirected + SMP duplicated edges: at most about `14.3 MiB` for `edge_index`
- Labels, time, masks: a few MiB total

A `16G` Slurm memory request is generous because pandas CSV loading has extra temporary overhead.

Submit with:

```bash
cd /home/jooyoungpak/workspace/GSMP-research
ELLIPTIC_ROOT=/path/to/elliptic_bitcoin_dataset sbatch run_elliptic_smp.slurm
```

You may need to edit `--partition=gpu` or uncomment the conda/module lines depending on your cluster.
